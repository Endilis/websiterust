Commands

Rust+ Commands

Manage your devices and monitor your server through Discord

.help
Info

Displays the full list of available commands with brief descriptions

Command list:

.help — Help
.pop — Server population
.time — Server time
.afk — AFK players
.mark — Position marker
.add — Add device
.remove — Remove device
.devices — Device list
.sw — Switch control
.events — Active events
.cargo — CargoShip status
.ch47 — CH47 status
.patrol — PatrolHeli status
.crate — Crate status
.deepsea — DeepSea status
.crash — Crash location
.lang — Chat language
.prefix — Chat prefix
.pop
Server

Shows current server population information

Displayed information:

  • 👥 Current number of players online
  • 🎮 Maximum slot count
  • ⏳ Queue size
💡 Tip: You can always quickly check online count and queue
.time
Server

Shows in-game time and countdown to sunrise/sunset

Time information:

  • 🕐 Current in-game server time
  • 🌅 Time until sunrise (minutes)
  • 🌆 Time until sunset (minutes)
💡 Tip: Plan your raids and farming knowing the exact server time — night won't catch you off guard
Important before using .add

Before adding a device with .add, you must first pair it in the Rust+ mobile app!

.add <device_name>
Device

Adds a smart device to the bot database for remote control

How to use:

  1. Pair the device in the Rust+ app and give it a name
  2. Run the command: .add device_name
  3. Receive a confirmation of successful addition
Example: .add main_light
💡 Tip: Use clear names without spaces — the shorter the word, the easier to type in an emergency
.devices
Device

Shows a list of all connected devices with their current state

Displayed data:

  • 📝 Device name
  • 📌 Device type
  • ⚡ Current state (on/off)
📌 Note: The list is output with a 1-second delay between devices
.remove <name | all>
Remove

Removes one or all devices from the bot database

Usage options:

.remove device_name

Removes a specific device

.remove all

Removes all devices

Examples:
.remove turret
.remove all
⚠️ Warning: .remove all will delete all your devices with no way to recover them!
.sw <device> <0|1>
Control

Remote control of smart switches

Command parameters:

device_name

Name of your device

1

Turn device on

0

Turn device off

Examples:
.sw turret 1 — Turn on
.sw turret 0 — Turn off
🎯 Pro tip: Use clear names without spaces for easy control
.afk
Command

Shows a list of teammates who have not moved for a long time

Displayed data:

  • 👤 Player name
  • ⏳ Idle time
💡 Tip: Teammate not responding in chat? Now you know they're not ignoring you — mom just called them for dinner)
.mark
Command

Sends your current map grid square to in-game chat and the Discord Alerts channel with a map crop

Displayed data:

  • 📍 Player name and map square (e.g. B7)
  • 📸 Map crop with red dot — sent to Discord Alerts channel
📌 Note: To send the screenshot to Discord, set up the Alerts channel in bot settings
🗺 Event commands

The following commands show the current status of in-game events: whether the event is active and in which map square.

.events
Events

Shows all currently active server events in one message

Tracked events:

  • 🚢 CargoShip — cargo ship
  • 🚁 CH47 — Chinook
  • 🚁 PatrolHelicopter — patrol helicopter
  • 📦 Crate — locked crate from helicopter
  • 🌊 DeepSea — deep sea event
💡 Tip: Use .events instead of individual commands — shows everything in one message
.cargo
Events

Shows the current status of the CargoShip event (cargo ship)

  • ✅ Whether CargoShip is active on the server
  • 🗺 Current map square
.ch47
Events

Shows the current status of the CH47 event (Chinook)

  • ✅ Whether CH47 is active on the server
  • 🗺 Current map square
.patrol
Events

Shows the current status of PatrolHelicopter (patrol helicopter)

  • ✅ Whether PatrolHeli is active on the server
  • 🗺 Current map square
.crate
Events

Shows the status of the Locked Crate — the crate dropped by a helicopter

  • ✅ Whether Crate is active on the server
  • 🗺 Current map square
.deepsea
Events

Shows the status of the DeepSea event (deep sea)

  • ✅ Whether the DeepSea event is active
  • 🌊 Underwater event status
.crash
Events

Shows the map square where the PatrolHelicopter crashed

How it works:

  • 🔍 Checks live markers for an explosion
  • 💾 If the marker disappeared — uses the last saved location from cache
  • 🗺 Returns the map square (e.g. D5)
💡 Tip: Use right after the crash — coordinates are cached even after the marker disappears
⚙️ Settings commands

Bot in-game chat settings. Changes apply to your entire guild.

.lang <ru|en>
Setting

Switches the language of bot responses in the Rust in-game chat for your guild

Usage options:

.lang ru

Switch to Russian

.lang en

Switch to English

Examples:
.lang ru
.lang en
.prefix <text>
Setting

Changes the prefix the bot adds to each message in in-game chat

Parameters:

  • 📝 Any text up to 20 characters
  • ⚠️ Default: [rustinfo.online]
Examples:
.prefix [BOT]
.prefix [MyServer]
💡 Tip: A short prefix is easier to read in in-game chat
Discord

Discord Commands

Commands for use in Discord channels

!map
Discord

Shows the server map with teammates' positions displayed

Displayed data:

  • 🗺 Full game server map
  • 👤 Positions of all teammates on the map
  • 📋 Map grid squares
💡 Tip: Use for quick orientation — full map with all team members' positions
!info <monitor name>
Discord

Shows the contents of a box or cupboard connected as a resource monitor

Displayed data:

  • 📦 Item list with quantities
  • 📋 Item icons
  • ⏳ Tool cupboard protection time (if applicable)
Example: !info my_box
📌 Note: The device must be added first using the .add command in in-game chat
Support

Frequently Asked Questions

Answers to common questions about RustInfo

How do I start using Rust+ commands?

To connect the bot, read the Setup guide.

Why don't commands work? .time .pop .add etc.

Make sure the bot is connected correctly and the message Connected was sent in chat

Did you remember to create the command?

The bot only works in team chat

The default command prefix is . (period)

How often is monitoring data updated?

Server data is updated every 10–30 seconds. Tracked player status is updated every 2–4 minutes to optimize server load.

Can I manage multiple servers?

Yes, RustInfo supports monitoring and managing multiple servers simultaneously. You can switch between them in Discord.

Is it safe to give the bot access to Rust+?

Yes, the bot uses the official Rust+ API and does not store your credentials. All connections are encrypted. You can revoke access at any time via the Rust+ app.

What if a device does not respond?

Check your Rust server connection. Make sure the device still exists in-game and is properly paired. Try removing it with .remove and adding it again.
